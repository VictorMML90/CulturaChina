# Actividad 3 – Gestión de Versiones e Integración del Prototipo

## Descripción
En esta actividad se documenta el proceso de gestión de versiones del prototipo, la exportación de recursos del diseño y el plan de integración con plataformas de desarrollo frontend.

---

## 1. Gestión de Versiones

Para la gestión de versiones del proyecto se utilizó **Git** junto con la plataforma **GitHub**, lo que permitió mantener un historial de cambios durante el desarrollo del prototipo.

El uso de control de versiones permite:

- Registrar cambios realizados en el proyecto.
- Colaborar entre los integrantes del equipo.
- Recuperar versiones anteriores del proyecto.
- Mantener organizado el desarrollo del prototipo.

**Repositorio del proyecto:**

El código fuente del prototipo se encuentra en este mismo repositorio, donde se gestionan las versiones mediante Git y GitHub.

---

## 2. Documentación de Cambios en el Prototipo

Durante el desarrollo del prototipo se realizaron diversas modificaciones para mejorar la interfaz y la experiencia del usuario.

### Cambios principales

- Creación de la estructura inicial de la página web utilizando HTML.
- Implementación de estilos con CSS.
- Adaptación del diseño para dispositivos móviles (diseño responsivo).
- Mejora del contraste de colores para accesibilidad.
- Implementación de botones interactivos para mostrar más información al usuario.
- Publicación del prototipo mediante GitHub Pages.

---

## 3. Exportación de Recursos del Prototipo

El diseño del prototipo fue realizado utilizando **Figma**.

Desde la herramienta de diseño se exportaron diferentes recursos necesarios para la implementación web.

### Recursos exportados

- Imágenes utilizadas en la interfaz.
- Iconos del diseño.
- Paleta de colores definida en el prototipo.
- Tipografía utilizada en la interfaz.

Estos recursos fueron integrados posteriormente en los archivos HTML y CSS del prototipo.

---

## 4. Plan de Integración con Plataformas de Desarrollo Frontend

Para integrar el prototipo con el entorno de desarrollo frontend se siguió el siguiente flujo de trabajo:

1. Diseñar la interfaz de usuario en Figma.
2. Exportar los recursos gráficos necesarios.
3. Implementar la interfaz utilizando HTML y CSS.
4. Subir el proyecto a un repositorio en GitHub.
5. Gestionar cambios mediante commits en Git.
6. Publicar el prototipo utilizando GitHub Pages.

Este flujo permite transformar el diseño del prototipo en una aplicación web funcional.
